神的编辑器or编辑器的神

看到一个老师的ppt介绍linux的编辑器，提到了Emacs和Vim，分别称为“神的编辑器，编辑器的神”，哈哈。

对我来说vim接触的比较早，但对大部分命令都不熟悉需要查表，而Emacs基础基本上为零；之前见过几个牛人都在用Emacs编辑，又想到网上说用好一款编辑器可以极大地提高工作效率，于是正式决定学一个编辑器。

效率

提到效率，比如说输入编辑特定的文档的效率，不考虑人为因素（输入速度、熟练程度等）。我们假设有一个固定的按键频率和思考速度，这样就把工作效率问题完全转换到对输入过程的控制的效率上。这时，工作效率当然和两个因素相关：
1 文档内容的熵
2 编辑器对编辑过程的定义和控制

在同样的字数下，熵比较小的文档显然能够快速输入，比如输入10个不同的字母快于输入10个相同字母；另一方面对于一篇正常的文档，如何采用最有效的输入流程？不同的软件处理方式也是有很大区别的，比如简单地从按键次数来定义不同的编辑器会有完全不同的表现。当然编辑过程很复杂，需要考虑多方面因素。理论上说，当编辑器的编辑控制方案设置好之后会存在一个最优的输入流程或者方案；但显然操作者很难找到它，而最终会选择自己熟悉的以及直觉的方案，从这个角度来说编辑效率实在是一件“人机合一”的东西。

pdfeditor-非常有用的pdf工具

在做ppt或者其他一些文档的时候我总会遇到一个问题：如何从pdf文件获取高质量的图片插入？
当我还是小白的时候，曾经用MATLAB画了几幅图，然后截图保存为图片（因为自动保存的图片往往带有难看的边框）插入到文档中，结果被老师狂批一顿。现在看来，这一问题最好的解决方法就是保存为矢量形式的图，不会损失信息；例如word里面有许多支持的矢量图格式，如wmf，eps等。

由于做ppt的需要，经常要引用一些论文的结果，这些论文都是pdf格式，如何高质量保存这些图呢？采用截图的方式显然不好，这样做出的文档稍微放大之后就会模糊，尤其是失去了数据图原本的矢量行为。经过一番摸索，我找到了一个比较合适的解决方案：pdfeditor。具体方法是：先用pdfeditor选取对象，保存为pdf文件，然后用$\LaTeX$导入到beamer或者tex文件中。这样获得的图形是无损的。

总结

pdf是一种简化的文档描述语言，所以对于一个图形来说，生成文件将会保持非常小的熵。如果截图的话就完全破坏了这种基于描述的秩序，而产生了对应的基于分布的秩序，显然不合适。pdfeditor的强大之处就在于它几乎是基于pdf源码操作的，也就是说你是直接编辑描述文档本身，这样的好处就是你可以按照需求随意调整参数以精确控制文档。

pdf is a good document format / description language!

黑苹果折腾记录

从6月26到7月3日，整整一个星期，终于把黑苹果装好了！除了显卡驱动不完美和USB驱动不能用，基本上达到了我的预期。现在我的win10和mac系统完美启动，爽歪歪！这期间经历了各种坑，各种折腾特；在此记录一下这次安装过程。

备份数据

关于黑苹果，本科的时候听说过但是感觉没什么用；去年冬天尝试装过一回，由于我一直是自己找帖子爬楼没有任何人的帮助和经验借鉴所以花了三四天也没成功。当时为了黑苹果专门买了块SSD，前几天拆电脑想起这事，加上最近也有时间就决定花上几天把黑苹果装好。
由于之前的折腾经历，我深深体会到“数据无价”。因此不管能否成功，都必须要保证现有数据的安全！为了以防万一我把SSD上现有的数据全部备份到移动硬盘（事实证明这很正确，之后我进行了重新分区），毕竟物理上的隔离最安全。

开始折腾

我的beamer模板

$\LaTeX$简介

$\LaTeX$是一个非常强大的排版系统，能够高效地制作出排版精美的文档。它和word之类的所见即所得排版软件的关系就类似于linux和Windows的关系。所以如果你需要高效地、精细地控制你的文字，请使用$\LaTeX$。

beamer简介

beamer是$\LaTeX$的一个用于presentation的类，语法和$\LaTeX$几乎一样。由于出色的公式排版，非常适合于学术类的内容演示。网上有很多模板，可以方便地修改并建立自己的模板。在熟悉beamer之后你会发现它几乎和$\LaTeX$一模一样，你甚至直接把论文中的东西粘过来就行。
