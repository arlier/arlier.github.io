pdfeditor-非常有用的pdf工具

发表于 2016-07-16   |   分类于 share

在做ppt或者其他一些文档的时候我总会遇到一个问题：如何从pdf文件获取高质量的图片插入？
当我还是小白的时候，曾经用MATLAB画了几幅图，然后截图保存为图片（因为自动保存的图片往往带有难看的边框）插入到文档中，结果被老师狂批一顿。现在看来，这一问题最好的解决方法就是保存为矢量形式的图，不会损失信息；例如word里面有许多支持的矢量图格式，如wmf，eps等。

由于做ppt的需要，经常要引用一些论文的结果，这些论文都是pdf格式，如何高质量保存这些图呢？采用截图的方式显然不好，这样做出的文档稍微放大之后就会模糊，尤其是失去了数据图原本的矢量行为。经过一番摸索，我找到了一个比较合适的解决方案：pdfeditor。具体方法是：先用pdfeditor选取对象，保存为pdf文件，然后用$\LaTeX$导入到beamer或者tex文件中。这样获得的图形是无损的。

总结

pdf是一种简化的文档描述语言，所以对于一个图形来说，生成文件将会保持非常小的熵。如果截图的话就完全破坏了这种基于描述的秩序，而产生了对应的基于分布的秩序，显然不合适。pdfeditor的强大之处就在于它几乎是基于pdf源码操作的，也就是说你是直接编辑描述文档本身，这样的好处就是你可以按照需求随意调整参数以精确控制文档。

pdf is a good document format / description language!

$\LaTeX$简介

$\LaTeX$是一个非常强大的排版系统，能够高效地制作出排版精美的文档。它和word之类的所见即所得排版软件的关系就类似于linux和Windows的关系。所以如果你需要高效地、精细地控制你的文字，请使用$\LaTeX$。

beamer简介

beamer是$\LaTeX$的一个用于presentation的类，语法和$\LaTeX$几乎一样。由于出色的公式排版，非常适合于学术类的内容演示。网上有很多模板，可以方便地修改并建立自己的模板。在熟悉beamer之后你会发现它几乎和$\LaTeX$一模一样，你甚至直接把论文中的东西粘过来就行。

我对薛定谔方程的理解

发表于 2016-06-25   |   分类于 物理

Schrödinger equation

著名的薛定谔方程：$$\color{orange}{ i\hbar\frac{\partial }{\partial t}\Psi =H\Psi }$$其中i是虚数单位，$i^{2}=-1$,$\hbar$是普朗克常量，$\Psi$是波函数，H是哈密顿量。如果你仔细分析这个方程，会发现她有很多神奇的地方。首先方程含有i，决定了这是一个复方程，也就是说你不可能用简单的实数去描述量子力学现象了；同时作为量子力学中的一个基本方程，包含$\hbar$是必须的。因为$\hbar$是描述量子现象最基本的物理量，是自然界最基本的物理“常量”之一，具有角动量的量纲。$\hbar$的大小是区分量子现象和经典现象的依据——根据玻尔的对应原理，当$\hbar$的大小趋近于零的时候量子力学就“退化”到经典力学。

蔡元培就任北大校长之演说

发表于 2016-06-25   |   分类于 随笔

许多针砭时弊之言能穿越时代，是作者的才气，也是时代的不争气 。
-卢昌海

蔡元培先生一百年前的演讲，于当今而言仍然适用。一百年之后我们民族在各个领域都取得了巨大的成就，然而就科学领域而言，我们近乎没有什么进步（这一点和日本对比就可以确证）。如果将科学视为一种文化，并将其追溯到文艺复兴甚至古希腊哲学，那么科学事实上和我们的传统文化完全不相容。我们了解并接受科学经历了一个漫长的过程，从翻译《几何原本》、《天演论》等著作到洋务运动以及后来的新文化运动。我们开始只想了解科学的形式和结果，后来才认识到科学是一种理念，有一些基本的精神：推理、演绎、质疑。也正是从新文化运动开始，我们的教育和西方的差距逐渐变小。三四十年代清华北大，南开等学校以及之后的西南联大的毕业生水平基本上属于世界一流（虽然在硬件上（实验设备、学校设施等）还有较大差距），这是巨大的成就。足见科学的核心在于创造知识的学者，以及他们的自由思想和科学精神而不是其他的客观条件。而今我们在科学领域的不足主要在于对科学精神的完全忽略，我们的建了很多大学，却没有明白大学的精神之所在：“非为做官，非为致富”。究其原因有历史和传统的因素，也有当今时代风气的影响；作为一个从事或者关注科学的个体，我们需要对科学保持最基本的鉴别能力，不做“误人误己”的事。

ROOT入门教程

发表于 2016-06-24   |   分类于 share

最近在学习ROOT，发现网上有一篇不错的入门教程。从最基本的概念和用法开始介绍，非常适合入门，因此翻译出来共享一下（虽然理论上接触ROOT的人英语肯定没问题）。文章是在百度文库找到的ROOT:a brief starting tutorial。

1.基础介绍

ROOT 是一个用于数据分析尤其是高能物理数据分析的面向对象的软件（OOF）。在面向对象的软件中有三个基础概念：类、对象和方法。类是对一个系统中某种一般性“东西”的描述，对象是类中的某一个具体的例子而方法则是类的函数。作为一个框架（framework），ROOT具有很多优点,比如更少的代码量（现有集成版本能够满足你的大多数需求，并且很稳定）和绘图、网络等服务。由于面向对象的设计ROOT允许对类进行扩展和修改来包含新的东西或者用同样的模型结构来描述实际问题的不同部分。
