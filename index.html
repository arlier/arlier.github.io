如何阅读文献

发表于 2016-09-08   |   分类于 share

《怎样阅读论文》
from：\url{http://ccr.sigcomm.org/online/files/p83-keshavA.pdf}

三次阅读法读文献

这一方法的核心在于将一篇文献阅读三次而不是一次从头读到尾，每一次阅读都建立在上一次的基础上并且有特定的目标：第一次是获得文章的主要观点；第二次掌握内容，但不是很具体；第三次深度理解文章。

第一次阅读只需要快速浏览，决定是否有必要继续阅读。这一次阅读大概5到10分钟即可，具体的步骤：

  • 认真阅读标题、摘要、引言
  • 阅度章节和子标题，忽略其它的内容
  • 阅读结论
  • 浏览参考文献，标出已经读过的

在第一次阅读完成后，你应该可以回答：

  • 这是那种类型的文章？
  • 它和哪些文章相关？分析和解决问题的理论基础是什么？
  • 作者的假设是不是有效的?
  • 这篇文章的主要贡献是什么？
  • 辨别这篇文章写的好不好？

根据上面这些信息，你就可以判断是不是需要进一步的阅读；同样的当你写一篇文章时，你应该预期读者仅仅会浏览一下你的文章，所以要谨慎地选择章节标题和小标题以及简洁的、易于理解的摘要和结论。

UNIX的哲学

发表于 2016-09-07   |   分类于 计算机

之前总是听说linux中“一切都是文件”的思想，今天看到了完整版的UNIX的哲学，分享一下。
The Unix philosophy is philosophical approaches to developing software based on the experience of leading developers of the Unix operating system. The following philosophical approaches also applies to Linux operating systems.

  • Do one thing and do it well - Write programs that do one thing and do it well. Write programs to work together. Write programs to handle text streams, because that is a universal interface.
  • Everything is file - Ease of use and security is offered by treating hardware as a file.
  • Small is beautiful.
  • Store data and configuration in flat text files - Text file is a universal interface. Easy to create, backup and move to another system.
  • Use shell scripts to increase leverage and portability - Use shell script to automate common tasks across various UNIX / Linux installations.
    Chain programs together to complete complex task - Use shell pipes and filters to chain small utilities that perform one task at time.
  • Choose portability over efficiency.
  • Keep it Simple, Stupid (KISS).

back_up_all_my_work

发表于 2016-08-23   |   分类于 随笔

备份

在用texide的时候，我突然意识到我可以用pdf文件备份我所有的重要工作；因为我可以使用include命令，来对不同的文章和内容进行独立管理和编辑引用，再把它们整合起来；然后结合书签命令进行快速分类和索引。如果实施成功，那么我将能够高效、安全地保护我的数据，尤其是这些低熵数据。同时由于我试图保存所有重要数据，文件的安全性，私密性值得考虑；同时还得详细确定如何组织这些内容，采用什么样的构架和管理形式，保证以后内容的添加和删除；要考虑的尽可能长远；初步打算，在完成模型之后先进行测试，如果没什么大问题的话就可以逐步把所有之前的文件进行集合。

标准

另一个问题是，确定一个明确的标准，即什么样的数据需要存储，什么样的数据不需要存储，或者只需要存储索引。然后就是怎么存储的问题，这个也需要明确的标准，对于存入的内容应该有具体的结构和标签之类的东西；总之要有统一的、确定的格式。怎么写着写着就感觉是在，用root进行数据存储。有点类似数据库的意思。

安全

我想在完全执行这一步骤之后，我的数据基本上就是安全的。我把这些低熵的数据在云端和网盘进行了同时备份；对于照片和视频之类的文件，丢了也不是很可惜。假设发生意外我的硬盘损坏，数据完全丢失，那我的损失也许并不大；可能很多重要的文件没了，但理论上其他人的电脑上是肯定会有的。比如我下载的文件、软件、照片等；我所需要的只是建立相应的索引和链接。

记录

1.临时工作目录，为知笔记随时记录想法和灵感，每周归档。
2.重要软件说明和索引，以及链接；对于可能失效的链接可以考虑备份文件到网盘。
3.重要第三方文件。
4.按照时间进行记录，然后添加分类和标签。
5.在开头进行格式说明，分类和标签列表；目录。
6.主文件全部采用include加注释形式，方便开关。

再次折腾-树莓派

发表于 2016-08-21   |   分类于 随笔

折腾电子制作历史


大一的时候在图书馆看到很多关于电子制作DIY，机器人，无线电制作等方面的书和杂志，之后就对电子制作产生了极大的兴趣；也做了很多很烂的作品。大二的时候开始玩51单片机，买了很多配件，但由于预算等原因也就是小打小闹；比如做了些流水灯，摇摇棒之类的；然后偶尔也试着做了无线充电，可见光通信，红外探测等一些略微需要技术的东西，虽然最后成功了，但很勉强。由于经常呆在演示实验室，电磁炮是玩的最多的，这东西需要的技术不多，但做的好的效果还是很不错的。之后由于考研和考试的压力，也就没有过多花时间在这上面，直到毕业的时候和学弟一起做了个速度计，算是比较满意的作品。细细想来我对硬件确实是有着某种痴迷，更准确的说是对某种确定性的痴迷，就好比你能隐约感觉到你怎也做就能达到神奇的效果，然后你就回努力去尝试。（一直一来的一个梦想就是有一个自己的工作室，能够3D打印，制作PCB，基本的材料加工工具，常用电子设备。。。。。不知道何时才能实现）现在来分析失败的原因，一是缺少制作PCB的能力和经验，因为但凡功能复杂一点的电路想要靠杜邦线连线实在太麻烦，所以最后都是测一些示例程序，基本上没什么自己的东西；二是资金不足，有很多想法仅仅是雏形，需要进行验证，这是需要成本的，当时考虑到后续要大量投入好多东西都放弃了。总之，等到条件成熟，一定组建一个自己的工作室！但愿那时候还不算晚。
