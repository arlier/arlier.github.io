PINE64内存卡更改分区大小

发表于 2016-12-26   |   分类于 计算机

昨天安装geant4的过程中发现磁盘不够用了，装了一天白费了；只能格式化内存卡进行重新分区。

调整PINE的内存卡分区

参考文章 http://blog.csdn.net/c80486/article/details/8460304

  • 查看分区信息 sudo fdisk /dev/mmcblk0 这是mmcblk0盘，有p0,p1两个分区
    p0是boot分区，p1是系统分区。
    我需要将p1的7个g改成30个g，因为我的内存卡是32g的。
  • 输入p，查看分区信息，记住linux分区p1的开始扇区位置，144860
  • 输入d再输入2，也就是删除linux分区
  • 输入n，新建分区，开始扇区选择144860（也就是linux的开始扇区），结束扇区选择默认的最大值
  • 输入w保存分区信息
    重启，执行sudo resize2fs /dev/mmcblk0p2，就可以看到新的分区了，这个方法相对简单一些。

在PINE64上安装G4的尝试

发表于 2016-12-25   |   分类于 计算机

开始安装


之前成功地在ubuntu上安装了G4，这次在Debian上安装应该会顺利一些，就是为了再熟悉一下安装过程。

本来我用的ubuntu的PINE的系统，可是后来执行apt命令网速很慢，然后卡死了网上找了很多办法都没法解决甚至开不了机，不得不重新更换系统。之前Debian安装很顺利，但是屏幕分辨率一直不对，弄了好久也不行；然而昨天我机智地改了/boot中的配置文件，然后就调成了1080p，非常好。据说Debian也比较稳定。对了，昨天ubuntu被玩坏就是因为我想安装一个3070的USB无线网卡，由于不能自动识别所以要编译驱动，结果编译了好几次都出错；然后在升级apt和linux内核的过程中系统死了。但是Debian是直接可以识别这块网卡的，而且网速非常快。

其实这就是在折腾，没什么明显的必要；我只是在尝试这种在非完整版系统上运行软件的可能性。就像当初内存卡无法识别一样，我发现是电脑的读卡器有一根针弯了，然后拨回来就能读出了；接着再用diskgenius重新分区重建MBR就好了。前几天我安卓手机只能识别出一个SIM卡，打开发现由于经常插拔造成卡槽的一根针断了，于是找了一根订书针接上了，也许这就是物理神奇的地方。一些law是如此稳定而普遍地成立，从而让你觉得这很美，对此坚信无疑。

apt-get被锁

发表于 2016-12-24   |   分类于 计算机

执行命令sudo apt-get update
显示：无法获得锁 /var/lib/apt/lists/lock - open (11: 资源暂时不可用)
搜索得知是因为意外退出apt导致的，于是sudo rm /var/lib/apt/lists/lock进行解锁就可以了。
